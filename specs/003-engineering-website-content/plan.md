## Summary

Implement Bhavya Dalwadi's portfolio content across Home, About, Projects,
Resume, and Contact as launch pages, then stage Case Studies as an immediate
follow-up page and Writing as optional. Use the existing style system and
component architecture, but update route-level section composition and content
sources so each block appears only where it belongs.

## Content Placement Guardrails

These rules are mandatory during implementation:

1. Home contains positioning, proof preview, and conversion prompts only.
2. About contains working style, technical focus, and fit narrative only.
3. Projects contains concise catalog and why-it-matters framing only.
4. Case Studies contains deep problem-role-approach-outcome narratives only.
5. Resume contains summary and download/contact actions only.
6. Contact contains outreach methods and role-fit note only.
7. Writing remains hidden until content threshold is met.
8. No section heading should appear on a different route unless explicitly
   defined in spec placement matrix.

## Implementation Phases

### Phase 0 - Inventory and Mapping

1. Audit existing routes and section components for overlap with new copy.
2. Produce a route-to-section mapping checklist directly from spec matrix.
3. Identify missing routes (resume, case-studies) and content schema gaps.

Exit criteria:
- Every source content block is assigned to exactly one route section.
- No ambiguous destination remains.

### Phase 1 - Launch Scope Content Integration

Launch pages: Home, About, Projects, Resume, Contact.

1. Home route
   - Implement Hero, Credibility Strip, Selected Work Preview, Engineering
     Strengths, Case Study Preview, About Preview, Contact CTA.
2. About route
   - Implement Intro, How I Work, Technical Focus, What I Look For,
     Beyond the Resume.
3. Projects route
   - Implement page intro, 6 featured projects, Earlier Work section.
4. Resume route
   - Implement intro, Snapshot bullets, Highlights, Download/Contact CTAs.
5. Contact route
   - Implement intro, Reach Out list, Best fit closing note.

Exit criteria:
- All launch pages show only their designated sections.
- Navigation includes Home, About, Projects, Resume, Contact.

### Phase 2 - Case Studies Follow-up Integration

1. Add Case Studies route with intro and 2 core studies.
2. Add optional WealthPilot section as an optional block with safe fallback
   when not published.
3. Wire Home and Projects case-study links to this route.

Exit criteria:
- Case Studies route is discoverable from nav and relevant CTAs.
- Deep narratives are not duplicated into Projects list cards.

### Phase 3 - Optional Writing Gate

1. Keep Writing out of navigation by default.
2. Add explicit publish gate rule: require at least 3 quality posts.
3. Enable route and nav item only when threshold is satisfied.

Exit criteria:
- Writing does not appear prematurely.

## Verification Strategy

### Integration Checks

1. Home section-order test validates all seven sections in order.
2. About section-presence test validates all five defined sections.
3. Projects test validates six featured projects and Earlier Work block.
4. Resume test validates download and contact CTAs.
5. Contact test validates outreach methods list and best-fit copy.
6. Case Studies test validates deep structure blocks if route is enabled.

### E2E Flow Checks

1. Home to Projects to Contact journey works without dead ends.
2. Home to Resume download CTA and Contact CTA are both reachable.
3. Case Studies links resolve correctly when route is launched.

### Content Correctness Checks

1. No Home-only section heading appears in About, Projects, Resume, or Contact.
2. No deep case-study copy appears in project card summaries.
3. Tone consistency spot-check across all launch pages.

## Risks and Mitigations

- **Risk**: Copy duplication across Home preview and full pages.
  - **Mitigation**: Use preview variants with strict character and purpose
    limits.
- **Risk**: Missing routes delay launch.
  - **Mitigation**: Prioritize launch pages first; treat Case Studies as
    immediate next increment.
- **Risk**: Navigation mismatch during staged rollout.
  - **Mitigation**: Gate nav items by route readiness flags.
